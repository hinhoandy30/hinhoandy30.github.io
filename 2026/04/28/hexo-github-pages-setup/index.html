你不需要买服务器、不需要买域名，完全免费。

日常写作流程（之后每次）

搞懂这几个概念

HTML 文件是做什么的，要不要管

完全不用管。 你只写 Markdown（.md），hexo generate 自动把它变成 HTML：

public/ 里的 HTML 是自动生成的，不需要也不应该手动修改。

主题 vs 文章模板，有什么区别

这是两个完全不同层面的东西：

控制什么 文件位置
主题 整个网站的视觉外观（布局、字体、颜色、导航栏） themes/ 目录
文章模板 新建文章时自动填入的默认字段 scaffolds/post.md

改主题，整个网站外观全变，你的文章内容不受影响。改模板，只影响下次 hexo new 时生成的 .md 文件头部默认有什么字段。

页面排版能用 Markdown 控制吗

Markdown 控制内容结构，主题控制视觉样式。

你写 ## 就是二级标题，写 ``` 就是代码块——这是内容结构，在任何主题下都有效。但最终标题多大、代码块什么颜色、文字间距多宽，全由主题的 CSS 决定，Markdown 改不了。

换主题，你的 .md 文件一个字不改，页面视觉效果就完全不同了。

为什么 GitHub 上没有 source 文件夹

hexo deploy 只把 public/ 里的 HTML 文件推送到 GitHub，不推送你的源文件：

GitHub Pages 只需要 HTML 来展示网站，不需要源文件。但这意味着如果电脑坏了，你的 .md 文章就丢了。建议另建一个仓库专门备份源文件。

关于”申请友链”

博客是纯静态的，没有后端，所以没有真正的”提交表单”功能。通常的做法是在友链页面写上联系方式（邮箱或 GitHub Issues 链接），让对方主动来找你。
